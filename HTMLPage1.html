Project description
  • The function "initialize" initializes the map and constructs the polygons that depict the CTAs that we used: EDUUFFM141, the red one, and EDUUFFM142, the green one. They are located around Frankfurt, Germany. The cordinates were taken from Germany's' AIP. After this, it calls to the other functions and the process starts.
  • The function "pathGeneration" generates a given number of flights that start and end at random ariports from Europe. It calls to "Trajectory".
  • The function "Trajectory" generates the intermediary points between the start and the end point based on the distance between them. If the distance is greater than 1000 KM, then the trajectory is computed as an orthodrome, otherwise it is a loxodrome.
  • The function "drawPaths" displays the trajectories computed using "Trajectory" on the map. It calls to "animate".
  • The function "animate" moves the markers, which represent the aircraft, along their trajectory. Using "window.setInterval" the function defined inside is repeated every 200 miliseconds. At the end, the function is terminated using "window.clearInterval" in order to prevent useless repetition of the same simulation. In order to start a second simulation the page must be reloaded. It calls to "inside".
  • The function "inside" checks whether a given point is inside a given polygon. It calls to "coordsOfAirspace".
  • The function "coordsOfAirspace" extracts the coordinates of the corners of a given polygon in order to ease their usage in the other functions.
